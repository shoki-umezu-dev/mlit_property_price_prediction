## 1. 背景・目的
- **ポートフォリオとして**
    - 目的: 国交省が公開している住宅価格のデータを活用して、特定エリアの価格トレンドを予測したい
    - 背景: 時系列データを扱えること、データの前処理、特徴量エンジニアリング、解釈性の高さを保持し、ビジネスの場を想定した分析が行えることをアピールしたい

- **ビジネスとして**
    - 目的: ビッグイベントである大阪万博の公表~実施が、大阪市の住宅価格にもたらした影響と傾向を分析し、IR開業時(2030年秋頃)に向けた不動産投資物件の選定を行う
    - 背景: 
        - メガイベントは周辺の物件全てを値上げするというよりは、値上がりする物件とそうでない物件で二極化が起こりやすい
        - 現在、大阪万博というビッグイベントを終えたからこそ、その恩恵をデータの視点から客観的に分析し、次に控えるIR開業に向けた物件の条件選定を行う必要がある

### Geminiが出した課題背景
大阪市場は2025年の国際的なメガイベント（万博）を経て、交通インフラの拡張や再開発が進み、不動産価格に大きな地殻変動が起きた。さらに2020年代後半（2030年秋頃）にはIR（統合型リゾート）の開所を控えており、投資家やデベロッパーにとって「次の成長エリア・物件」の選定は極めて重要な意思決定となっている。

しかし、公開されている取引データ（〜2025年）には、特殊な取引特有のデータ欠損（用途や事情の空欄）や、駅徒歩「30分〜60分」といった郊外・湾岸部特有の表記の揺れが存在し、これらが価格形成に与える影響は定量化されていない。

本プロジェクトでは、これらの不動産固有のデータ特性（欠損・ノイズ）を適切に前処理・特徴量化し、万博前後の市場トレンドから「イベントの投資対効果の波及ロジック」を解明する。その上で、2026年現在のマクロ的な価格予測を行うとともに、将来のIR実施に向けて『大阪で真に値上がるポテンシャルのある物件条件（都市計画・容積率・駅距離等によるスクリーニング構造）』をデータから導き出すことを課題とする。


## 2. ターゲットユーザー・ユースケース
- **想定ユーザー**: 不動産投資家、エリア選定を行うデベロッパー、物件購入を検討している個人
- **ユースケース**: IRに向けて、どのような物件が値上がりする傾向にあるのか、買い時なのかを判断するため


## 3. 分析対象データ
- **データソース**: 国土交通省「不動産取引価格情報」オープンデータ「中古マンション等」
- **対象エリア**: 大阪府大阪市
- **対象期間**: 2014年第三四半期~2025年第四四半期
- **主なデータ項目**: 
- **意図背景**
    - 中古マンションデータを活用する理由: 宅地や農地などは土地の形や接道状況など、個体差が激しくデータの傾向としてトレンドを出すことが困難な可能性が高いと考えたため
    - エリアが大阪府大阪市である理由: 
        - 候補としては取引が盛んな東京都、住み慣れた千葉県があったが、東京都は常に再開発がされており、価格が局所的に高騰しているかつ「港区は価格が高い」といった当然の結論しか出ない可能性がある
        - 千葉県は再開発などのトレンドによる影響は出るかもしれないが、ローカル過ぎてクライアント側の想像が付きずらかったり、マーケットが狭過ぎて投資対象になりにくい
        - 大阪は万博やIRといったトレンドがあり、IRはまだ到来していないため、未来予測に対する面白みがある
    - 対象期間について: 
        - 万博やIRを公表する前から、開発の進み具合などに応じてトレンドを分析するため
        - 大阪維新のかいが構想を発表した2014年8月からのデータをもとに分析を行う


## 4. 機能要件/分析タスク
- **[F-1] データ収集・前処理**
    - 欠損値処理、文字列データの数値化など
- **[F-2] EDA**
    - 各特徴量と目的変数の傾向の可視化
    - ラグを作成しトレンド、季節性、残差に分けて分析を行う
- **[F-3] 価格予測モデルの構築**
    - 時系列モデル(Prophetなど)や回帰モデル(EBM, lightGBM)を用いた未来の価格を予測
- **[F-4] 分析結果の可視化**
    - Tableauを用いた、投資判断用のダッシュボードの構築


## 5. 非機能要件
- **再現性**
    - 誰が実行しても同じ結果が出るよう、データ前処理からモデル学習までのパイプラインをコード化
- **可観測性**
    - 分析の成果物として、Tableauでの視覚的レポートを提供し、非エンジニアでも直感的に理解可能にする
- **解釈性**
    - ビジネスの現場を想定し「なぜその予測結果になったのか」を説明できる「解釈性」を損なわないような分析を行う


## 6. tool/アピールポイント
### 使用ツール・技術スタック
* **データ抽出・加工 (ETL):** SQL (Snowflake または PostgreSQL/BigQueryを想定)、Python (Pandas)
* **モデリング・解釈 (ML):** InterpretML (EBM: Explainable Boosting Machine)、Scikit-learn
* **可視化・レポート (BI):** Tableau Desktop
* **環境構築・構成管理:** Git, GitHub

### 本プロジェクトにおけるアピールポイント
1. **「解釈性（ホワイトボックス）」を最優先した技術選定**
   ビジネス現場での説明責任（ROIの証明）を担保するため、あえてブラックボックスな手法（通常のDeep Learningやチューニングのみを重視したLightGBM）を避け、モデル自体が完全な説明性を持つEBM（Explainable Boosting Machine）を採用。各変数の非線形な影響を厳密に可視化できるスキルを証明。
2. **ドメイン知識（不動産）を活かした「時間軸のハック（ラグ特徴量）」**
   単なる時間の経過ではなく、万博やIRのアナウンスメントが2年、3年のタイムラグを経て不動産価格に波及するメカニズムをモデル化。マクロ経済イベントと不動産固有の遅行性を紐解く特徴量エンジニアリングを実践。
3. **実務を意識した「SQL ✕ Python ✕ Tableau」のデータパイプライン**
   データの一次加工（集計・結合・基礎的な前処理）をSQLで行い、機械学習の実験をPython、最終的な経営層へのプレゼンをTableauで行うという、モダンなデータ現場の標準ワークフローを単独で一気通貫して再現。


## 7. スケジュール
* **第1週: データ基盤構築 ＆ データの「泥臭い」一次加工（SQLフェーズ）**
  * 国交省CSV（11年分・73,000件）のデータベースへのインポート。
  * SQLを用いたデータの結合、欠損フラグの作成、駅徒歩「30分〜60分」の数値置換処理　など。
* **第2週: ドメイン特徴量の作成 ＆ データの可視化（EDAフェーズ）**
  * Python（Pandas）を用いた「ラグ特徴量（1〜3年前のインバウンド数など）」の作成。
  * 2014年〜2025年の時系列トレンド、エリアごとの価格二極化の傾向をプロット。
* **第3週: EBMモデルの構築 ＆ 精度・解釈性の評価（モデリングフェーズ）**
  * 時系列分割（Time Series Split）を用いた、データリークのない評価環境の構築。
  * EBMの学習を実行し、2025年データでの精度検証、および各変数の寄与度の抽出。
* **第4週: 2026年予測の出力 ＆ ダッシュボード化（BI・ドキュメントフェーズ）**
  * 予測結果と解釈性データをTableauに連携し、インタラクティブなダッシュボードを構築。
  * README（ポートフォリオ）の整理、成果物の最終確定。


## 8. 分析前仮説
- 大阪万博による影響は「公表による宣伝効果(時系列)」「開催地域へのアクセスの良さ(エリア)」「インバウンド効果(エリア、観光地としての魅力度合い)」「インフラ整備や都市開発が伴ったかどうか(エリア、時系列)」が影響すると考える
- 公表による宣伝効果: 
    - 各エリアの価格が急騰したタイミングの前3年の間で大阪万博に関する宣伝があったかどうか、ドメインを参照しながら分析を行う
- アクセスの良さ: 
    - 最寄り駅の情報から、乗り換え回数や所要時間を辞書データを作成し、マッピングする
- インバウンド効果:
    - 大阪府や官公庁が公開している「大阪府の外国人宿泊者数」の年次データなどをネットから拾ってきて、取引年ごとに横から結合する
    - その年の大阪市内の観光名所ランキングデータを引っ張ってきて横から結合する
- 都市開発フラグ: 
    - 大阪万博に伴う湾岸道路の整備、夢洲の拡張、日本館の設立、淀川ゲートウェイ、淀川左岸線、GREEN×EXPO2027などの開発があったエリアにフラグを設ける
